# Ad Funnel Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a complete, responsive HTML/CSS design for the Facebook ad funnel — 8 views in a single-page flow with hash-based routing — that the web app team will integrate into the Keywords.app web app.

**Architecture:** Single HTML file (`funnel-start.html`) with all 8 funnel views. JavaScript handles hash-based routing and view transitions. Light theme, responsive (mobile-first). Image placeholders use styled boxes with descriptions. This is a design deliverable — the web app team adapts the HTML/CSS into their app framework.

**Tech Stack:** Static HTML/CSS/JS. Google Fonts (DM Serif Display + Figtree). No build tools or frameworks.

**Spec:** `docs/superpowers/specs/2026-04-05-ad-funnel-design.md`
**Pricing reference:** `docs/keywords-app-pricing-spec.md`
**Responsive reference:** cascader.io — clean light design, frosted-glass header, generous whitespace, slate/blue palette (we use green accents instead)

---

## File Structure

One file:
- **Create:** `funnel-start.html` — complete funnel with all views, routing, CSS, and JS

Each task adds a section to this file. Task 1 creates the file with the shell (CSS variables, routing JS, responsive framework). Tasks 2–9 each add one view's HTML inside the shell. Task 10 adds final polish and transitions.

## Chunk 1: Shell and Landing Page

### Task 1: Create funnel shell with routing and responsive framework

**Files:**
- Create: `funnel-start.html`

This task creates the complete file skeleton: CSS variables, global styles, responsive breakpoints, hash-based routing JS, and view transition logic. All subsequent tasks add HTML inside the `<main>` element.

- [ ] **Step 1: Create `funnel-start.html` with the complete shell**

Use the `frontend-design` skill to create the file. The file must include:

**CSS requirements:**
- Light theme. CSS variables for the color system:
  - Background: `#fafafa`, Cards: `#ffffff`, Borders: `#e4e7ec`
  - Text hierarchy: dim `#8b919e`, mid `#5f6574`, default `#2d3039`, dark `#1a1c22`
  - Green accent: `#2d9a49`, light `#38b558`, bg `rgba(45,154,73,0.06)`
- Google Fonts: DM Serif Display (headings) + Figtree (body) — Figtree is the web app's primary font
- Mobile-first responsive: base is mobile, `min-width: 768px` for tablet, `min-width: 1024px` for desktop
- Max content width: 1060px centered
- Frosted-glass header bar: logo left, "Log in" link right, sticky, backdrop-blur

**HTML structure:**
```html
<header><!-- sticky frosted header: logo + login --></header>
<main>
  <section id="view-landing" class="funnel-view active">...</section>
  <section id="view-signup" class="funnel-view">...</section>
  <section id="view-connect" class="funnel-view">...</section>
  <section id="view-pricing" class="funnel-view">...</section>
  <section id="view-select" class="funnel-view">...</section>
  <section id="view-discovery" class="funnel-view">...</section>
  <section id="view-scanning" class="funnel-view">...</section>
  <section id="view-ready" class="funnel-view">...</section>
</main>
```

**JS requirements:**
- Hash-based routing: maps `#signup` → `view-signup`, `#connect` → `view-connect`, etc.
- Default (no hash or `#`) shows `view-landing`
- `showView(viewId)` function: hides all `.funnel-view` sections, shows the target one with a fade-in
- Listen to `hashchange` event
- `navigateTo(hash)` helper that sets `window.location.hash`
- All CTA buttons use `onclick="navigateTo('#signup')"` etc.
- Browser back button works because we're using real hash changes

**View sections:** For this task, add placeholder content in each section — just the `<section>` tags with a centered heading showing the view name (e.g., "Landing Page", "Sign Up"). Subsequent tasks replace these placeholders with full content.
